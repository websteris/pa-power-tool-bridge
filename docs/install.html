CLI Bridge Installation

Allow scripts, Claude, and other tools to read and write your Power Automate flow definitions without copy-paste — all via simple local files.

Requirements

Node.js 16+ Required to run the host process. Install from nodejs.org or via nvm / Volta.
Extension ID Find it at chrome://extensions or edge://extensions with Developer mode enabled — or in the extension's Help panel.

Install

Windows
macOS
Linux

Run this one-liner in PowerShell. The installer will prompt for your extension ID.

powershell -ExecutionPolicy Bypass -Command "irm https://raw.githubusercontent.com/websteris/pa-power-tool-extension/main/install.ps1 | iex"

The bridge installs to %LOCALAPPDATA%\pa-power-tool\ and registers a native messaging host for both Chrome and Edge.

Run this one-liner in Terminal, replacing <extension-id> with your ID.

curl -fsSL https://raw.githubusercontent.com/websteris/pa-power-tool-extension/main/install.sh | bash -s <extension-id>

The bridge installs to ~/.local/share/pa-power-tool/ and registers a native messaging host for Chrome, Edge, and Brave.

Run this one-liner in your terminal, replacing <extension-id> with your ID.

curl -fsSL https://raw.githubusercontent.com/websteris/pa-power-tool-extension/main/install.sh | bash -s <extension-id>

The bridge installs to ~/.local/share/pa-power-tool/ and registers a native messaging host for Chrome, Edge, and Brave.

After installation

The bridge connects automatically within a few seconds — no browser reload needed. The status indicator in the extension's Help panel will turn green when connected.

Files are written to your system temp directory:

current-flow.json Full flow definition, updated on open and save
status.json Live state, errors, and last saved time
commands.json Write a command here to control the extension

Windows: %TEMP%\pa-power-tool\  ·  macOS: $TMPDIR/pa-power-tool/  ·  Linux: /tmp/pa-power-tool/

Send commands

PowerShell
'{"command":"save"}' | Set-Content "$env:TEMP\pa-power-tool\commands.json"
bash / zsh
echo '{"command":"save"}' > "$TMPDIR/pa-power-tool/commands.json"
getJsonWrite current editor content to current-flow.json
saveSave the current flow to Power Automate
runRun with last inputs
reloadReload flow definition from the server
setJsonReplace editor content — include "json" field with the new definition
setJsonAndSaveReplace content and immediately save to Power Automate

Troubleshooting

  • If the bridge doesn't connect, reload the extension from chrome://extensions or edge://extensions.
  • Check the log file for errors: %TEMP%\pa-power-tool\host.log (Windows) or $TMPDIR/pa-power-tool/host.log (macOS/Linux).
  • If Node.js was installed after the bridge, re-run the installer so the host can locate it.
  • Open an issue on GitHub if you're still stuck.
